    入力できる方程式は自励形に限ります(非自励形については、現在確認中)。

<p class="pb10">
    When you click [to Plot], the following screen will be displayed.
    You can enter any equation and check its solution trajectory. <br>
    When you press the [START] button, the equations you entered will be displayed in TeX format below the input box,
    and the drawing will begin.
    If you get an error message, please check and correct the entered equation as necessary. <br>
    The equations that can be entered are limited to autonomous systems (non-autonomous systems are currently under review).
</p>
<p class="pb30">
    Below are some input examples.
</p>
<ol class="fwm pb40, pl40">

      <p class="pb10">
          Example of a 2-dimensional first-order linear autonomous system.
      </p>
      <p>

      </p>
    </li>
    <li class="pb40">
      <p class="pb10">
          Example when there is no need to form a system of equations. Let $dx/dt$ be 1 .
      </p>
      <p>
          <img src="image/DEQ_img/deq4_case2a.jpg">
      </p>
    </li>
    <li class="pb40">
      <p class="pb10">
          Various inputs are possible for powers, but for simplicity, x^2 is sufficient. Of course, x*x is also fine.
          For squares, x*x is fine, but for higher powers like fourth or fifth, it's easier to input as x^4, etc. <br>
          You can also use pow().
          When pow(x^2 + y^2, 2) is displayed in TeX, it appears as $(x^2 + y^2)^2$ without any issues,
          but if you input pow(x, 5), it will display as $(y)^5$. The calculation itself will be correct, so there is no problem,
          but if it feels awkward, simply input y^5.
      </p>
      <p>
          <img src="image/DEQ_img/deq4_case3a.jpg">
      </p>
    </li>
    <li class="pb40">
      <p class="pb10">
          If the equation becomes long,
          you can input it as shown below (however, if it is excessively long, the layout of the screen may be disrupted.
          Although this will not affect the calculation, it will be very difficult to read).
          Also, if a part of the equation is repeated, you can input the repeated part in the where, $r = $
          and replace the repeated part in the equation with r to save input effort.<br>
          If not needed, there is no need to input anything like in examples 1 to 3.

    <li class="pb40">
      <p class="pb10">
          Since math.js is used to parse the entered formulas,
          mathematical functions such as sin(), cos(), and pow() can be entered as they are.<br>
          Conversely, if a formula including math.pow(), math.sin(), math.cos(), etc., is entered in the input box,
          it will result in an error, and the drawing will not be executed.<br>
          For exponents, e^x is evaluated as $e^x$.
          If you input exp(x), it will be displayed as $\exp(x)$ in TeX, but it will be evaluated in the same way.
          If the exponent is complex, please use parentheses, like e^(x + 1) or exp(x + 1).
      </p>
