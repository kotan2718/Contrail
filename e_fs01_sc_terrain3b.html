An easy method making Terrain Scenery

Guiana Highland

introdaction

As for the geographical feature data of the Microsoft Flight Simulator (MSFS), the "digital elevation data" is adopted after the FS2000.
A certain area is divided into the mesh at constant intervals of 30m or 90m.
Next, the pin which shows an average altitude to the center of each mesh is stood.
It shows the geographical feature data of 3D like pins on a pincushion.

The drawing engine of the MSFS expresses the "geographical features data of 3D" by putting the texture on the tip of each pin of this pincushion, and making it smoothly connect.
The MSFS calls this geographical features data "terrain".

In FS2000, because it takes much time to drawing the geographical features in this method, it hardly became practical use.
However, the method of drawing the "terrain" data has been greatly improved from FS2002.
It came to be able to fly very comfortably on complex geographical features even when the "terrain" built based on super-detailed data of 30m mesh was used.

However, in spite of having carried such drawing engine, a 1km mesh terrain data is used in most areas of the MSFS.
As for this, the same is said of FS2004.
Of course, if a mesh is made fine, the amount of data will also increase exponentially.
Therefore, I think that there is a limit in the quality of data.

The MSFS opened "Terrain SDK (Software Development Kit)" to the public along with the adoption of "Terrain" data.
If the data along SDK is made, "Terrain" data can be used with add-on scenery.
The compiler tool for that was offered.

However, various formats exist in the digital elevation data.
If the data that "Terrain SDK" directs is used, the terrain data can be made with a compiler tool.
It is necessary to convert the digital elevation data offered by other formats into the format specified by the "Terrain SDK" to use the compiler tool.

In those days, as for the data specified by the "Terrain SDK", only the data of U.S.A. was being opened to the public.

Recently, the digital elevation data that the space shuttle had collected was open to the public.
Now, this data is covering almost all the world.
It is called SRTM (Shuttle Radar Topography Mission) data.
The 90m data (3arc/sec) and the 30m data (1arc/sec, the U.S.A. only) can be acquired as a freeware.

Since the SRTM data agrees in the format specified by the MSFS, it can create the "terrain" data directly.
However, a fatal data-loss exists in the SRTM data.
It is the datas in altitude 0 of the SRTM data.
I will call this data loss the "missing data".
I leave the discussion about the "missing data" to the specialists.
In this page, the wonderful tool which interpolates this "missing data" and creates the "terrain" data is introduced.

The wonderful tool is SRTMtoBGL by Mr. Matthew Styles.
This tool converts the SRTM data into the terrain data for the MSFS.
The "missing data" is interpolated at the same time.

You will experience a wonderful flight by witnessing the detailed geographical features.

How to make the "terrain" data

== NOTE ==
Please make the "terrain" data in your own responsibility.

[1] The SRTM data:

The SRTM data:

http://dds.cr.usgs.gov/srtm/

Refer to the following page for the details of SRTM.

http://www2.jpl.nasa.gov/srtm/

[2] Mr. Matthew Styles's the "SRTMtoBGL" tool

The file name is srtm_to_bgl_104.zip.

The purpose file can obtain from flight Simulation-related various sites.
I introduce the download method about Flightsim and avsim , please refer to it.

Now, all data was assembled.

Normally, apart from this tool, the compile tools offered by "terrain SDK" are required.
However, those compiler tools are also enclosed by srtm_to_bgl_104.zip.

[3] Making of the "terrain" data

The "srtm_to_bgl_104.zip" is compressed by the zip form.
Make a folder.
The folder name is arbitrary. Here, let's set the folder name to [SRTMtoBGL].
Copy the "srtm_to_bgl_104.zip" to [SRTMtoBGL] folder.
Decompress the "srtm_to_bgl_104.zip" by "winzip" etc. in the folder.

The srtm data is also compressed by the zip form.
Copy the decompressed data to [SRTMtoBGL] folder.

Refer to the "README.TXT" of srtm_to_bgl_104 for the subsequent works.
It is a concise manual.
The terrain data is completed at a few minutes if it follows the instruction of the manual.

The file name which the "srtm_to_bgl_104" outputs is "SRTM_3ARCSEC.bgl", if the srtm data is [3arc/sec].
Change the file name suitably as it is comprehensible later.

[4] The registration as add-on scenery to the MSFS

Copy the "*.bgl" file to the [Addon Scenery] - [scenery] folder of FS2004.

Start FS2004 and move to the applicable area.
Probably, you will see the detailed geographical features which was not able to be expressed in 1km mesh.

The following images are Guiana Highlands in South America.
The left image is being drawn with 1km mesh of the MSFS default.
The right image is being drawn with 90m mesh of the SRTM data.

I wanted to fly over the Table Mountains in Guiana Highlands when I could obtain the detailed geographical features data.
In 1km mesh, the data of geographical features is only a gentle hill. However, in 90m mesh it is changed dramatic.

A scenery made from the above-mentioned explanation is not necessarily perfect.
A design of the coastline may be needed in an area including the seashore.
However, if it restricts to the geographical feature data of a terrain base, it can create easily in this way.
The following photograph is terrain scenery (3 arc/sec) of the Kerama islands in Okinawa
(since in addition to the improvement of terrain, the coastline is also improved, this scenery is a reference work).

When you make the detailed terrain scenery of the area in which you live, the MSFS will change like another simulator.
