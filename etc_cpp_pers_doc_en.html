Plotting Surfaces Tool   User's Manual

Plotting Surfaces Tool
User's Manual

This tool (hereafter referred to as "Pers") visualizes the graph (surface) of a function defined by z = f(x, y) using hidden lines processed meshes (red lines : z $\ge$ 0, blue lines : z $<$ 0). I have released a similar tool as a web version. The main feature of the web version is to select a function prepared in advance from the list and plot the surface. It also has the abilities to plot user-defined functions, but it has not been possible to save or load defined functions. On the other hand, Pers enhances the features that allow users to define their own functions, and provides the ability to save and load defined functions.

[e] Plotting Range Settings

Set the plotting range for x, y, and z.

The default range is -2.0000 to 2.0000 for x, y, and z. *note

After changes, press the [Start] button to replot the surface.

For the z-axis, if the function's values exceed the range, a plane parallel to the x-y plane will be drawn. To make the boundaries clearer, contour lines are drawn at three points: z : max, 0, min. When the gamma angle (the vertical viewpoint position) will be explained in [g] Plotting Angle Settings is set to 90 or -90 degrees, a graph of the implicit function will be plotted.

Fig. 1-6

[f] Mesh Settings

Set the number of lines used for drawing. "calculation" refers to the number of lines used for calculation. "drawing" refers to the number of lines actually drawn.

Increasing the number of lines makes the mesh denser and the surface smoother, but it also increases the computational load, which may cause choppy movements when rotating the rendered surface.

The setting range is "1 to 120". In practice, "20" or "40" is probably appropriate. If the setting value for "calculation" is changed, it will be automatically copied to "drawing". The initial value for both "calculation" and "drawing" is 20. *note

The "calculation" setting is copied to "drawing", so there is usually no need to change it. However, if you would like to intentionally reduce the number of lines to be drawn, set it to 1/2 or 1/4 and so on of the value set in "calculation".

After changes, press the [Start] button to redraw the surface.

[g] Plotting Angle Settings

"alpha" sets the horizontal angle (around the z-axis) of the viewpoint. "gamma" sets the vertical angle (downward [-] / upward [+]) of the viewpoint.

Although you can directly enter numerical values in the text boxes,

Although you can directly enter numerical values in the text boxes, the right slider of the drawing window links to "alpha" and the bottom slider links to "gamma", so moving the sliders will display the respective angles. The default is "alpha" : 30 degrees and "gamma" : -30 degrees. *note, (refer to [m] Plotting Angle Setting (Slider bar)).

After changes, press the [Start] button to redraw the surface.

After checking the box, press the [Start] button will output the log.

Fig. 1-7

[i] Discontinuity Points Replacement ON/OFF

When calculating z-values along a mesh, the denominator might become zero, or both the numerator and denominator could become zero.

In such cases, the result of the calculation will be a non-number such as "inf", "-inf", "nan", or "-nan(ind)". When the graph is drawn, those parts will be missing, as nothing is drawn.

Fig. 1-8

In these cases,, if you check [discontinuity points replacement], the discontinuity points (lines) will be replaced as follows:

Calculation results Replacement
inf
greater than 9999.0		 9999.0
-inf
less than -9999.0		 -9999.0
-nan(ind) result of calculation of one previous coordinate
nan result of calculation of one previous coordinate

When the the [START] button pressed , a message will appear showing the discontinuities point (line) and the replacement contents. However, since there can be a large number of replacements, it can be annoying to see the message displayed every time, so the message is only displayed once. Also if there are multiple causes of discontinuity, they will all be only displayed once. For more detailed information, refer to [h] Log Output ON/OFF.

Fig. 1-9

Fig. 1-10

This replacement is not perfect, and there may be cases where discontinuity occurs due to causes that I have not yet noticed. If you encounter such functions, please let me know. If I can deal with it, I will improve it.

[p] Perspective Settings (draw parameters: L, D)

In general, as one moves the viewpoint closer to an object, the area closer to the viewpoint becomes larger and the area farther away becomes smaller, creating a sense of perspective.

Conversely, if your viewpoint is far enough away from the object, there will be little perspective.

D is the distance from the drawing window to the object (surface), and is fixed at "10" as a specification.

L is the distance from the viewpoint to the drawing window, and is set to "1000" by default. In this case, there is almost no perspective effect and the surface is drawn with oblique projection.

Since Pers is intended to be drawn with oblique projection, the fixed values of L=1000, D=10 are fine. But perspective effects are taken into account in the design of Pers, so the value of L can be changed.

The range that can be set is from 1000 to 10.

Once you have made your settings, press the [START] button. The settings will be reflected in the surface rendering. If you set L to between 20 and 10, you can immediately see the perspective effect.

However, since the perspective effect is not the purpose of Pers, it is usually not necessary to change L.

User defined function formula is evaluated using TinyExpr. Formulas that can be expressed as text, such as (x^2 + y^2), will work without issues. The following operators and mathematical functions can be used when defining a function formula.

Operators

You can use the four arithmetic operations "+, -, *, /" and power "^". However, for square roots "√" and absolute value symbols "|…|", use sqrt() and abs(), respectively.

Mathematical functions

Usual functions such as sin, cos, exp, log, and pow can be used. The following functions can be used (functions that are not related to drawing surfaces have been omitted):
abs, acos, asin, atan, atan2, cos, cosh, e, exp, gamma, ln, log, log10, pow, sin, sinh, sqrt, tan, tanh

For the correspondence with the C and C++ standard function "math.h" library, refer to [04] About input errors.

When you press the [CHECK] button, the formula will be evaluated, and if the formula is valid, the surface of the entered function will be drawn and the [SAVE] button will appear.

Fig. 2-03

If there are errors in the evaluation of the entered formula, a message will be displayed, but the function used may be undefined, so check it (refer to [04] About input errors).

Fig. 2-04

Once you have corrected any errors in the formula, press the [CHECK] button again. If the corrections have been made successfully, the screen shown in "Fig. 2-03" will appear. If you find that undefined mathematical functions (eg., asinh, acosh) are being used, since it is not possible to register formulas, press the [CANCEL] button to exit user-defined mode. Pressing the [CANCEL] button clears the entered data and returns the screen to its initial state.

Before pressing the [SAVE] button, here is an overview of the settings that will be written to the file.

range

Set the drawing range for x, y, and z. By default, x, y, and z are all between -2.00000 and +2.00000, but they can be set individually. For details, refer to [e] Plotting Range Settings in Chapter 1 ([Ⅰ] About the screen).

draw parameters

You can set the alpha angle (rotation around the z-axis) and gamma angle (looking down or up to the x-y plane) from the viewpoint. For details, refer to [g] Plotting Angle Settings in Chapter 1 ([Ⅰ] About the screen).

alpha angle : -180 to 180 deg.
gamma angle : -90 to 90 deg.

line number

Sets the number of lines that make up the mesh to be plotted. For details, refer to [f] Mesh Settings in Chapter 1 ([Ⅰ] About the screen).

calcration : Specifies the number of lines to calculate.
drawing : Specifies the number of lines to plot. Normally, the number set by "calcration" is set, so there is no need to change it.

If you change any of the settings, press the [CHECK] button to check the plotting.
You can press the [CHECK] button any number of times.

[03] A function formula with parameters

The function you define may include parameters "a" through "d".

If the formula includes the letters "a" through "d", pressing the [Tab] key or clicking in the comment field after entering the formula will enable the "parameters" input boxes corresponding to those letters.

example:

formula: (x^2 + y^2 + a^2)^2 - 4*a^2*x^2 - b^4

comment: Cassini oval

This function can draw surfaces of various shapes by assigning different values to parameters "a" and "b". The contour line at z = 0 is called a "Cassini oval".

[01] The plotting window color settings

For the drawing window, you can set the color of the background, mesh lines, and x, y axes. However, the settings are not made from the screen, but by editing the initial file.

Open the [Pers] folder in Explorer. Inside is the [dat] folder. When you open the [dat] folder, you will find a file called "Pers_color.ini". Edit this with a text editor.

The contents of "Pers_color.ini" are as follows:

Fig. 3-01

Empty lines and lines beginning with "//" are ignored.

The following five lines are valid as data:

block title data
window back color window_back (0,0,0)
line color z value plus domain line_plusk (255,0,0)
line color z value minus domain line_minus (0,0,255)
axis x color axis_x (255,255,0)
axis y color axis_y (0,255,255)

The five lines of valid data are the colors that have been used in the previous explanations. They are also the colors that will be used as defaults if the data in the ini file is not loaded correctly.

In each block, the lines beginning with "//" are included as samples. If you remove the "//", the color of that data will be loaded.

Of course, you can freely set the data in RGB format. However, do not change the format of the data line.

format：title=(0-255,0-255,0-255)   ==> i.g. window_back=(185,185,185)

<01> About the disorder of hidden line processing

Pers was designed with a strong focus on hidden line processing in order to plot curved surfaces with meshes, but in some cases, lines that should not be visible are plotted. In the first figure shown in this user's manual (Fig. 1-1), there are some disorders on the left side.

Fig. 3-02

I would like to consider this issue as a task for the future.

<02> Problem with the power symbol "^" when entering a formula

Pers evaluates the entered formula using TinyExpr.

However, if the expression contains "-x^2" or "-y^2," the evaluation will be incorrect.
These terms carry a negative sign, involve a power symbol, and their exponent is an even number such as 2 or 4.

For example, when x=2, TinyExpr outputs "4" as the evaluation result of "-x^2".

If the input contains "-x^2", etc., replace it with "-(x^2)"or"-x*x".

Currently, this is the only evaluation error I am aware of. There is a possibility that evaluation errors may occur in other patterns. If so, please let me know.
