地形データについて

Microsoft (R) Flight Simulator(FS) の地形データは FS2000 からdigital elevation data が 採用されています。 ある地域を 30m とか 90m など一定間隔でのメッシュで区切り， 各メッシュの平均標高をその位置の経緯を組み合わせると， 針山が並んだような３次元の地形データとなります。
FS の描画エンジンは，この針山の先にテクスチャを貼り付け， 滑らかに接続させることで，３次元の地形データを表現しています。 FS はこの方法で作成した地形データを terrain と表現しています。

FS2000 では terrain での地形描画に時間がかかり， フレームレートが低すぎてほとんど実用になりませんでした。 しかし FS2002 からは，terrain データの描画方法が大幅に改善され， 30mメッシュという超詳細なデータを基に築かれた terrain データを用いた場合でも， 非常に快適に複雑な地形の上を飛ぶことが出来るようになりました。

しかし，このような描画エンジンを搭載したにもかかわらず， FS2004 ですら，FS デフォルトの多くの地域の terrain データは 1km メッシュという荒い標高データが採用されています。 もちろん メッシュを細かくして行くと，そのデータ量も エクスポーネンシャルで増加しますので， ユーザーサイドが FS に一方的に詳細なデータの提供を求めるというのも， 無理があるかもしれません。

FS は，terrain データを採用するにあたり， アドオン・シーナリーで利用できるように そのデータ作成方法を "terrain SDK" で公開し， そのためのコンパイラ・ツールを提供しています。

ところが，標高データには様々なフォーマットが存在します。 "terrain SDK" が指示しているデータを用いれば， コンパイラ・ツールを用いて，terrain データを作成することが出来ます。
それ以外の，フォーマットで提供される標高データは， 一旦，FS の "terrain SDK " で指示するフォーマットに変換しなければ， コンパイラ・ツールを利用することが出来ないのです。 "terrain SDK" が指示している詳細なデータは， 当時は合衆国のデータだけでした。

最近，スペースシャトルを利用した (約)90m メッシュの標高データが公開されました。 現在，ほぼ世界中のデータが網羅されています。
SRTM（Shuttle Radar Topography Mission ) データと呼ばれるものです。 ここでは，90m データ (3arc sec）と， 30m データ（1arc sec これは合衆国のみです）を フリーウェアとして入手することが出来ます。

SRTM データは，FS が指示するデータ・フォーマットに合致するため， SDK の指示に従えば，Terrain データを作成することが出来ます。 しかし SRTM データには，致命的なデータロスが存在します。 正式にはどのように呼ばれているのか分からないのですが， 高々度の山岳地帯を中心にデータの至るところに 標高ゼロの穴 が開いているのです。
ここでは，このデータロスを missing data と呼ぶことにします。 どうして missing data が存在するのかは ここで議論しても始まりませんので， この missing data を補間し，terrain データを作成する "すぐれツール" を紹介致します。

"すぐれツール" とは，Matthew Styles の SRTMtoBGL です。 このツールを利用することで，必要な SRTM データを入手すれば， 瞬時に，FS の terrain データに変換され，missing data も補間されて， 高詳細な地形データで描画された場所を飛行することが出来るようになります。

terrain データの作成方法

これから説明する terrain データの作成については，自己責任において実行して下さい。

[1] SRTM データの入手先

http://dds.cr.usgs.gov/srtm/

SRTMの詳細は下記のページを参照して下さい。

http://www2.jpl.nasa.gov/srtm/

[2] Matthew Styles氏の SRTM -> bgl 変換ツールの入手

ファイル名は，srtm_to_bgl_104.zip です。

Flight Simulation 関連の様々なサイトから入手可能ですが，
URL http://Flightsim.com/ と http://www.avsim.com/ については， ダウンロード方法 を紹介していますので，参照して下さい。

本来であればこのツールとは別に，"terrain SDK" で提供された コンパイルツールが必要になるのですが，
srtm_to_bgl_104 には（然るべき理由があるのだと思いますが）， それらのコンパイラツールも同梱されています。

これで，全てのデータが揃いました。

[3] terrain データの作成

srtm データは zip形式で圧縮されていますので，解凍します。
解凍したデータを srtm_to_bgl_104.zip を解凍したフォルダーにコピーします。

ここからの作業は，Srtm to BGL 104 の "README.TXT" を参照して下さい。
簡潔かつ平易な取説です。
指示通りに操作すれば，数分で terrain データが出来上がります。

仕上がったファイルは，3arcsec のデータならば， "SRTM_3ARCSEC.bgl" となります。 ファイル名を後で分かりやすいように適宜変更して下さい。

[4] 作成した terrain データの上を飛ぶ

[3] で作った bgl ファイルを Flight Simulator 2004 の "Addon Scenery" - "scenery" フォルダーにコピーします。

FS2004 を起動し，該当地域に移動します。 1km メッシュでは表現されなかった世界が，そこには広がっているはずです。

以下の画像は，FSデフォルトの1km メッシュと srtm 90m メッシュで作成した南米のギアナ高地です。

高詳細な地形データが入手出来たら一度飛んでみたいと思っていた秘境です。 デフォルトの 1km メッシュでは，なだらかな丘に過ぎない地形が， 90m メッシュでは劇的に変化します。

上記の操作で，FS の addon シーナリーが完成したわけではありません。
海岸を含む地域では海岸線の作成が必要になるかもしれません。

しかし，terrain ベースの地形 (標高) データに限れば， このように簡単に作成することが出来ます。

次の写真は，慶良間諸島の 90m メッシュです（この画像はterrainの改善に加えて，海岸線も改善されています。 従って参考作品です）。

あなたが住んでいる地域の詳細な地形が再現出来たならば， FS は今までとは全く異なったシミュレーターに変身するでしょう。
